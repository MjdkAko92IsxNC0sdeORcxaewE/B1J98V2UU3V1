## Attacker Path

**preconditions:**
- The TORUS/TITAN_X Uniswap V3 pool has `observationCardinality == 1` (the protocol's `addLiquidityToTorusTitanXPool` never calls `increaseObservationCardinalityNext`, so the pool retains the Uniswap V3 default of 1). [5](#0-4) 
- With cardinality 1, `observations[(observationIndex+1) % 1]` always resolves to `observations[0]`, whose timestamp is the previous block's timestamp (~12 s ago on mainnet). `getOldestObservationSecondsAgo` therefore returns ~12 s.
- The intended 15-minute TWAP window (`secondsAgo = 15 * 60 = 900`) is silently replaced by `secondsAgo = 12` because `if (oldestObservation < secondsAgo) secondsAgo = oldestObservation` has no minimum-window floor. [6](#0-5) 

**attacker-controlled inputs:**
- A large sell of TORUS into the TORUS/TITAN_X pool in block N-1, depressing the TORUS/TitanX tick.
- A call to `swapTitanXForTorusAndBurn(deadline)` in block N (or MEV-bundle ordering to ensure it lands after the manipulation).

**call sequence:**
1. Block N-1: Attacker sells TORUS → TitanX, pushing the pool tick down (TORUS cheaper). The observation written at the start of block N-1 captures the pre-manipulation tick; the observation written at the start of block N captures the post-manipulation tick.
2. Block N: `swapTitanXForTorusAndBurn` is called (by anyone, including the attacker).
   - `getTorusQuoteForTitanX` calls `getOldestObservationSecondsAgo` → returns ~12 s.
   - `consult(pool, 12)` computes a TWAP over the single 12-second window, which reflects the depressed tick from the attacker's block-N-1 dump.
   - `expectedTorusAmount` is low; `adjustedTorusAmount = expectedTorusAmount * 90 / 100` is even lower.
   - `amountOutMinimum` is set to this near-spot, manipulated value.
3. Block N (same tx or MEV bundle): Attacker front-runs the protocol swap by buying TORUS (pushing price back up), then the protocol swap executes at the inflated price and receives fewer TORUS than fair value (but passes the weakened `amountOutMinimum` check). Attacker back-runs by selling TORUS, capturing the spread.

## Why Existing Checks Fail

**No minimum TWAP window floor.** The code unconditionally replaces `secondsAgo` with `oldestObservation` when the pool is young or has cardinality 1, with no `require(secondsAgo >= MIN_ACCEPTABLE_WINDOW)` guard. [7](#0-6) 

**`consult()` only rejects `secondsAgo == 0`.** A 12-second window passes the `require(secondsAgo != 0, "BP")` check and proceeds to compute a near-spot TWAP. [8](#0-7) 

**10% slippage tolerance is insufficient.** `titanXToTorusSlippage = 10` (owner-settable down to 2%) means the attacker only needs to move the TORUS/TitanX price by >10% in one block to make `amountOutMinimum` non-binding. Given the pool's concentrated liquidity and the protocol's large swap sizes (hundreds of billions of TitanX per interval), this is achievable with realistic capital. [9](#0-8) 

**Protocol never increases observation cardinality.** `addLiquidityToTorusTitanXPool` mints the LP position but does not call `pool.increaseObservationCardinalityNext(N)`, leaving the pool permanently at cardinality 1 unless an external party does so. [5](#0-4) 

## Rejection Checks

**expected behavior checked:** The 15-minute TWAP fallback to `oldestObservation` is documented as a safety measure for new pools, but the protocol's own pool is the target — it is not expected to remain at cardinality 1 indefinitely. The silent collapse to a 12-second window is not documented as acceptable behavior.

**prior report checked:** No prior report found in the indexed codebase.

**README/NatSpec checked:** No NatSpec on `getTorusQuoteForTitanX` acknowledges the cardinality-1 risk or documents a minimum acceptable window.

**unsupported assumption checked:** The attack does not require impossible oracle values or malicious token owner action. It requires only that the pool retains its default cardinality (confirmed by absence of any `increaseObservationCardinalityNext` call in the codebase) and that the attacker has sufficient TitanX/TORUS capital to move the price >10% in one block.
