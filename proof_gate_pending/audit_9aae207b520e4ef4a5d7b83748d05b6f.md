## Attacker Path

**preconditions:**
- `liquidityAdded == true` (pool seeded).
- No caller has invoked `swapTitanXForTorusAndBurn` for N × `INTERVAL_TIME` (10 min) seconds. The attacker does not need to actively block anyone; they simply wait and race to be first after a long dormant period.
- `totalTitanXBurn > 0` (TitanX has been distributed for burning).

**attacker-controlled inputs:**
- Timing of the call (passive: just wait).
- `_deadline` parameter (any valid future timestamp).

**call sequence:**
1. Wait until `block.timestamp - lastBurnedTitanXIntervalStartTimestamp > N * INTERVAL_TIME` for large N (≥ 7199 intervals ≈ 50 days to hit the cap with `DAILY_ALLOCATION_TITAN_X_BURNING = 200` bps).
2. Call `swapTitanXForTorusAndBurn(_deadline)`.
3. The `burnTitanXIntervalUpdate` modifier fires `_intervalUpdateTitanXForBurning()`, which calls `_calculateIntervalsTitanXBurn(timeElapsed)`.
4. `_totalAmountForInterval` is set to `min(amountPerInterval * (1 + missedIntervals), totalTitanXBurn)`.
5. After the cap triggers, `amountAllocated = totalTitanXBurn`.
6. `incentive = totalTitanXBurn * 150 / 10_000 = 1.5% * totalTitanXBurn` is transferred to `msg.sender`. [3](#0-2) [4](#0-3) 

## Why Existing Checks Fail

**The only guard is the `totalTitanXBurn` cap** (line 598–600). It prevents the allocation from exceeding the pool balance, but it does not bound the incentive relative to a single interval's entitlement.

**Arithmetic of the cap trigger** (using live values: `DAILY_ALLOCATION_TITAN_X_BURNING = 200` bps, `INTERVALS_PER_DAY = 144`):

```
amountPerInterval = totalTitanXBurn * 200 / 10_000 / 144
                  = totalTitanXBurn / 7200

cap triggers when: amountPerInterval * (1 + missedIntervals) >= totalTitanXBurn
                   (1 + missedIntervals) >= 7200
                   missedIntervals >= 7199  →  ~50 days of inactivity
```

After 50 days, `amountAllocated = totalTitanXBurn` and `incentive = 1.5% * totalTitanXBurn`.

**Single-interval incentive** = `1.5% * (totalTitanXBurn / 7200) ≈ 0.000208% * totalTitanXBurn`.

**Maximum incentive after cap** = `1.5% * totalTitanXBurn` — a **7200× amplification** over the per-interval entitlement.

From live state: `totalTitanXBurn = 390,047,518,149,437,255,354,831,991,687` raw units (~390 billion TitanX). Maximum extractable incentive ≈ **5.85 billion TitanX** in a single call.

The `IntervalAlreadyBurned` check (line 231) only prevents replay within the same interval slot; it does not limit accumulation across missed slots. [5](#0-4) 

## Rejection Checks

**expected behavior checked:** The accumulation pattern is intentional (incentivize liveness), but the 7200× amplification to the full pool cap is not documented as intended. No NatSpec or comment bounds the per-call incentive to a single interval's share.

**prior report checked:** No prior report found in the repository.

**README/NatSpec checked:** No NatSpec on `_calculateIntervalsTitanXBurn` or `swapTitanXForTorusAndBurn` documents the missed-interval accumulation as a deliberate design choice with this magnitude.

**unsupported assumption checked:** The attacker does not need oracle manipulation, flash loans, or any privileged role. The only requirement is passive waiting and being first to call after a long dormant period — fully reachable by any EOA.

    function _intervalUpdateTitanXForBurning() private {
        if (block.timestamp < startTimeStamp) revert NotStartedYet();
        uint32 timeElapsed = lastBurnedTitanXIntervalStartTimestamp == 0
            ? uint32(block.timestamp - startTimeStamp)
            : uint32(block.timestamp - lastBurnedTitanXIntervalStartTimestamp);
        uint32 _lastInterval;
        uint128 _amountAllocated;
        uint32 _missedIntervals;
        uint32 _intervalStart;
        bool updated = false;
        if (lastBurnedTitanXIntervalStartTimestamp == 0) {
            (
                _lastInterval,
                _amountAllocated,
                _missedIntervals
            ) = _calculateIntervalsTitanXBurn(timeElapsed);
            _intervalStart = startTimeStamp;
            updated = true;
        } else if (timeElapsed > INTERVAL_TIME) {
            (
                _lastInterval,
                _amountAllocated,
                _missedIntervals
            ) = _calculateIntervalsTitanXBurn(timeElapsed);
            _intervalStart = lastBurnedTitanXIntervalStartTimestamp;
            updated = true;
            _missedIntervals++;
        }
        if (updated) {
            lastBurnedTitanXIntervalStartTimestamp =
                _intervalStart +
                (_missedIntervals * INTERVAL_TIME);
            titanXIntervalsBurn[_lastInterval] = IntervalBurn({
                amountAllocated: _amountAllocated,
                amountBurned: 0
            });
            lastTitanXBurnIntervalNumber = _lastInterval;
        }
    }
