## Attacker Path

**preconditions:**
- TitanX/TORUS pool has low liquidity (or is recently created with low observation cardinality, so `oldestObservation < 900`).
- Attacker holds sufficient TitanX and TORUS capital (or flash-loan access) to move the pool price.
- A non-zero `amountAllocated` exists in the current `titanXIntervalsBurn` interval (i.e., `distributeTitanXForBurning` has been called and the interval is ready).

**attacker-controlled inputs:**
- Timing of the attack (warp/wait until TWAP reflects the manipulated price).
- Pool trades to move spot price downward (sell TORUS → buy TitanX) for the TWAP window duration.
- `_deadline` parameter passed to `swapTitanXForTorusAndBurn`.

**call sequence:**
1. Attacker sells TORUS for TitanX in the TitanX/TORUS pool, depressing the spot price.
2. Attacker waits for `secondsAgo` seconds (up to 900s, or less if `oldestObservation < 900`). The TWAP accumulates the depressed price.
3. Attacker calls `swapTitanXForTorusAndBurn(_deadline)`.
4. Inside `_swapTitanXForTorus`, `getTorusQuoteForTitanX` reads the manipulated TWAP → returns a deflated `expectedTorusAmount`.
5. `amountOutMinimum = deflated_quote × 90%` — a very low floor.
6. The Uniswap router executes the swap at the current (still-depressed) spot price, delivering far fewer TORUS than fair value. The swap passes the `amountOutMinimum` check.
7. The TORUS received is burned. The attacker, who pre-sold TORUS at fair value, buys it back cheaply after the protocol swap pushes price back up, capturing the spread.

## Why Existing Checks Fail

**`oldestObservation` fallback shrinks the TWAP window:** [3](#0-2) 

If `oldestObservation < 900`, `secondsAgo` is reduced to `oldestObservation`. In the extreme case (e.g., pool cardinality = 1, `oldestObservation` = a few seconds), the TWAP collapses to near-spot and is trivially manipulable within a single block or a few blocks.

**Slippage tolerance is the only guard, and it is set to 10%:** [4](#0-3) [5](#0-4) 

`titanXToTorusSlippage = 10` (owner-settable, default 10%). If the TWAP is manipulated to 50% of fair value, `amountOutMinimum` becomes 45% of fair value — the swap succeeds while the protocol receives less than half the fair TORUS output.

**No spot-vs-TWAP divergence check:** There is no guard comparing the TWAP quote to the current pool `slot0` price or any external reference. A large divergence between TWAP and spot (the hallmark of manipulation) is not detected.

**`swapTitanXForTorusAndBurn` is permissionless:** Any address can call it once per interval, so the attacker controls the timing of the protocol swap relative to their manipulation. [6](#0-5) 

## Rejection Checks

**expected behavior checked:** TWAP-based slippage protection is the intended design, but the `oldestObservation` fallback and fixed 10% tolerance are insufficient when the pool is new or illiquid. This is not documented as an accepted risk.

**prior report checked:** No prior report reference found in the repository.

**README/NatSpec checked:** No NatSpec on `getTorusQuoteForTitanX` or `_swapTitanXForTorus` acknowledges TWAP manipulation risk or low-cardinality pool behavior.

**unsupported assumption checked:** The attack requires low liquidity or low observation cardinality — a realistic condition for a newly seeded pool. The live context confirms `liquidityAdded = true` and the pool was seeded with `INITIAL_TITAN_X_FOR_LIQ = 200_000_000_000e18` TitanX and `INITIAL_LP_MINT = 20_000e18` TORUS. [7](#0-6) 

The TORUS supply is small relative to TitanX, making the pool potentially thin on the TORUS side and susceptible to price impact.

    function _swapTitanXForTorus(
        uint256 amountTitanX,
        uint256 _deadline
    ) private returns (uint256 _torusAmount) {
        titanX.approve(UNISWAP_V3_ROUTER, amountTitanX);
        bytes memory path = abi.encodePacked(TITAN_X_ADDRESS, POOL_FEE, address(torusToken));

        uint256 expectedTorusAmount = getTorusQuoteForTitanX(amountTitanX);
        uint256 adjustedTorusAmount = (expectedTorusAmount * (100 - titanXToTorusSlippage)) / 100;

        ISwapRouter.ExactInputParams memory params = ISwapRouter
            .ExactInputParams({
                path: path,
                recipient: address(this),
                deadline: _deadline,
                amountIn: amountTitanX,
                amountOutMinimum: adjustedTorusAmount
            });
        return ISwapRouter(UNISWAP_V3_ROUTER).exactInput(params);
    }
```

    function getTorusQuoteForTitanX(
        uint256 baseAmount
    ) public view returns (uint256 quote) {
        address poolAddress = torusToken.titanXTorusPool();
        uint32 secondsAgo = 15 * 60;
        uint32 oldestObservation = OracleLibrary.getOldestObservationSecondsAgo(
            poolAddress
        );
        if (oldestObservation < secondsAgo) secondsAgo = oldestObservation;
        (int24 arithmeticMeanTick, ) = OracleLibrary.consult(
            poolAddress,
            secondsAgo
        );
        uint160 sqrtPriceX96 = TickMath.getSqrtRatioAtTick(arithmeticMeanTick);
        quote = OracleLibrary.getQuoteForSqrtRatioX96(
            sqrtPriceX96,
            baseAmount,
            TITAN_X_ADDRESS,
            address(torusToken)
        );
    }
```
