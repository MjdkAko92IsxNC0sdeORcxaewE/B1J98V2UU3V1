## Why Existing Checks Fail

`releaseFractalFunds()` has only a time-gate check and no caller restriction: [8](#0-7) 

The design intent is that `swapTitanXForTorusAndBurn()` releases fractal funds **after** the interval allocation is snapshotted (line 249–251). The `burnTitanXIntervalUpdate` modifier runs first and reads `totalTitanXBurn` at that moment. Because `releaseFractalFunds()` is public, an attacker can call it in the same block before the swap, causing the modifier to see the post-release (inflated) `totalTitanXBurn`. The internal call at line 249 then finds `lastFractalRelease` already updated and skips the release — so the inflation is permanent for this interval.
