## Attacker Path

**preconditions:**
- The TORUS/TITAN_X pool has `observationCardinality = 1` (Uniswap V3 default; never expanded). With cardinality 1, `getOldestObservationSecondsAgo` returns `block.timestamp - observations[0].blockTimestamp`, which equals the seconds since the last swap in the pool. If the last swap was recent (e.g., 30–60 seconds ago), `secondsAgo` is capped at that value instead of 900.
- Attacker holds TORUS tokens sufficient to move the pool price.

**attacker-controlled inputs:**
- Timing of their own swaps (to control what the short TWAP reflects).
- Calling `swapTitanXForTorusAndBurn` as the trigger transaction.

**call sequence:**
1. Attacker sells a large amount of TORUS for TITAN_X in the pool, pushing the TORUS spot price down (fewer TORUS per TITAN_X in the TWAP).
2. Attacker waits for the short TWAP window (e.g., 2–5 blocks at cardinality 1) to fully reflect the depressed price.
3. Attacker front-runs `swapTitanXForTorusAndBurn`: buys TORUS with TITAN_X, pushing the spot price back up.
4. Attacker calls `swapTitanXForTorusAndBurn`. `getTorusQuoteForTitanX` uses the short, manipulated TWAP → returns a low `expectedTorusAmount` → `adjustedTorusAmount` (= `amountOutMinimum`) is low. The swap executes at the inflated spot price without reverting.
5. Attacker back-runs: sells TORUS at the inflated price, capturing the sandwich profit.

## Why Existing Checks Fail

The fallback at line 1128 is the root cause:

```solidity
if (oldestObservation < secondsAgo) secondsAgo = oldestObservation;
``` [4](#0-3) 

When `observationCardinality = 1`, `oldestObservation` equals the seconds since the last pool swap. An attacker who controls when swaps occur can ensure this is small (e.g., 30 seconds = ~2 Ethereum blocks). A 2-block TWAP is trivially manipulable.

The 10% slippage guard (`titanXToTorusSlippage = 10`) only applies as a percentage of the already-manipulated TWAP quote, not of the true fair price:

```solidity
uint256 adjustedTorusAmount = (expectedTorusAmount * (100 - titanXToTorusSlippage)) / 100;
``` [5](#0-4) 

If the TWAP is manipulated to reflect 50% of the true price, `amountOutMinimum` becomes 45% of fair value — a 55% window for the sandwich. The `nonReentrant` guard does not prevent multi-transaction sandwich attacks across blocks.

## Rejection Checks

**expected behavior checked:** The fallback to `oldestObservation` is intentional for new pools, but the pool has been live since July 2025. The vulnerability is not about pool age — it is about cardinality remaining at 1 indefinitely, which is the Uniswap V3 default and is never expanded in this contract.

**prior report checked:** No evidence of a prior report in the indexed codebase.

**README/NatSpec checked:** No NatSpec on `getTorusQuoteForTitanX` documents the cardinality-1 risk or acknowledges that the TWAP window can be shortened to seconds.

**unsupported assumption checked:** The attack does not require impossible oracle values or malicious token owner action. It requires only that the pool's observation cardinality is 1 (the default) and that the attacker can time their swaps — both are realistic on mainnet.

## Local Proof Required

**test type:** Foundry fork test pinned to a recent Ethereum mainnet block.

**test file to add:** `test/TWAPManipulation.t.sol`

**test setup:**
1. Fork mainnet at a recent block.
2. Read `torusToken.titanXTorusPool()` to get the pool address.
3. Call `IUniswapV3Pool(pool).slot0()` to confirm `observationCardinality`.
4. If cardinality = 1: simulate a large TORUS→TITAN_X swap to depress the TWAP, then `vm.roll` forward 3 blocks, then call `getTorusQuoteForTitanX` and assert the returned quote is significantly below the current spot price.
5. Simulate the sandwich: front-run buy, call `swapTitanXForTorusAndBurn`, back-run sell. Assert attacker's net TITAN_X balance increased.

**expected assertion:**
```solidity
assertLt(quote_after_manipulation, fair_quote * 0.7e18 / 1e18); // TWAP < 70% of fair
assertGt(attacker_titanX_after - attacker_titanX_before, 0);    // attacker profits
```

**failure condition:** If `observationCardinality > 1` and the oldest observation is older than 900 seconds, the full 15-minute TWAP is used and the manipulation cost becomes prohibitive — the finding would be rejected. The local test must confirm cardinality = 1 on the live pool to validate the precondition.
